# Difference-Objects-Classes

# Overview
This project demonstrates the conceptual differences between classes and objects in Java. It includes clear explanations, a simple Java example, and comments within the code to highlight how classes act as blueprints while objects represent real instances created from those blueprints.

# Definitions

# Classes
A class is a blueprint or template that defines the structure and behavior of something in a program. It specifies the attributes (variables) and behaviors (methods) that its future objects will have.
A class itself does not occupy memory as a real entity—it simply describes what an object could be.

# Objects
An object is an actual instance created from a class. It exists in memory, holds real data, and can perform actions defined by the class.
Objects created from the same class share the same structure but contain their own unique values.

### Conceptual Differences
| Class | Object |
| ----- | ------ |
| Blueprint or template | Real instance created from the blueprint |
| Defines attributes and behaviors | Has actual attribute values and can perform behaviors |
| Does not exist in memory until used | Exists in memory when instantiated |
| One class can create many objects | Each object is unique, even if created from the same class |
